# Amanflix Update Helper

Automates production version updates by swapping code folders while preserving the virtual environment, database, and all user data.

**The problem:** Previously, updating the production server meant copying the entire project (1.5TB of videos + 68K poster images), taking ~15 hours. The update helper separates code from data, so updates only swap the lightweight code folder — taking under a minute.

---

## How It Works

### Architecture

The app's source code was modified to read all data paths from a central config file instead of using hardcoded relative paths:

```
api/data_config.json  →  api/paths.py  →  all source files import from paths.py
```

- **`config/data_config.json`** — Contains a single `data_root` key pointing to the external Data folder
- **`paths.py`** — Reads the config and exports path constants (`UPLOADS_DIR`, `CDN_FILES_DIR`, etc.)
- If `data_config.json` is missing, all paths fall back to the original relative locations (backward compatible for development)

### Folder Structure (Production)

```
Z:\Amanflix\
├── Data\                      ← permanent data (never touched during updates)
│   ├── uploads\               ← user-uploaded video files (~1.5TB)
│   ├── files\                 ← JSON metadata (movies/tv catalogs)
│   ├── posters_combined\      ← poster images (~68K files)
│   ├── instance\              ← SQLite database (amanflix_db.db)
│   ├── backups\               ← database backups
│   └── logs\                  ← application logs
│
├── Amanflix_prod\              ← current production code
│   ├── api\                   ← Flask backend (contains paths.py)
│   │   ├── config\            ← config folder
│   │   │   ├── data_config.json   ← points to ..\..\..\Data
│   │   │   └── service_config.json
│   │   ├── venv\              ← Python virtual environment (preserved across updates)
│   │   └── paths.py           ← central path module
│   └── repo\                  ← React frontend
│
├── Amanflix_new_version\       ← downloaded new code (placed here before running update)
├── Amanflix_backups\           ← archived old production versions
│   ├── backup_2026-04-01_14-30-00\
│   └── ...
│
├── update_helper.py            ← this script
└── update_config.json          ← generated by 'configure' command
```

## Commands

### `python update_helper.py configure`

Interactive setup wizard. Run this once to set all paths. Generates:
- `update_config.json` — paths used by the update helper itself
- `data_config.json` — placed inside `Amanflix_prod/api/` for the Flask app

**Prompts:**

| Prompt | Description | Example |
|--------|-------------|---------|
| Data folder path | Where the permanent data lives | `Data` |
| Production folder | Current production code folder | `Amanflix_prod` |
| New version folder | Where to place downloaded updates | `Amanflix_new_version` |
| Backup folder | Where old versions are archived | `Amanflix_backups` |
| Venv path | Relative to production folder | `api/venv` (default) |

### `python update_helper.py update`

Performs a version update in 6 steps:

| Step | Action | Details |
|------|--------|---------|
| 1/6 | Stop app.py | Finds and kills the running Python process |
| 2/6 | Extract venv + config | Moves `venv/` to a temp location, reads `data_config.json` |
| 3/6 | Archive old prod | Moves `Amanflix_prod/` → `Amanflix_backups/backup_YYYY-MM-DD_HH-MM-SS/` |
| 4/6 | Activate new version | Renames `Amanflix_new_version/` → `Amanflix_prod/` |
| 5/6 | Restore venv + config | Moves venv back into new prod, writes `data_config.json` |
| 6/6 | Start app.py | Launches app.py using the venv's Python interpreter |

**Prerequisites:** Place the new version code in `Amanflix_new_version/` before running.

### `python update_helper.py status`

Shows:
- Current configuration values
- Path validation (checks if each expected folder/file exists)
- Whether app.py is currently running (with PIDs)
- List of available backups with sizes

### `python update_helper.py rollback`

Restores a previous version from backups:

1. Lists all available backups (newest first)
2. You pick which one to restore
3. Current prod is archived as `pre_rollback_YYYY-MM-DD_HH-MM-SS`
4. Selected backup is copied to production
5. venv and data_config.json are preserved (same as update)
6. App is restarted

## Generated Files

### `update_config.json`

Created by `configure`. Used by the update helper script itself.

```json
{
    "data_folder": "Data",
    "prod_folder": "Amanflix_prod",
    "new_version_folder": "Amanflix_new_version",
    "backup_folder": "Amanflix_backups",
    "venv_path": "api/venv",
    "data_root_relative": "..\\..\\Data"
}
```

### `data_config.json`

Lives inside `Amanflix_prod/api/config/`. Read by `paths.py` at app startup.

```json
{
    "data_root": "..\\..\\Data"
}
```

The `data_root` value is a relative path from `Amanflix_prod/api/` to `Data/`. The `configure` command calculates this automatically.

## Path Mapping Reference

How the old hardcoded paths map to the new external Data folder:

| Old Path (relative to `api/`) | Data Folder Subfolder | `paths.py` Constant |
|-------------------------------|----------------------|---------------------|
| `uploads/` | `Data/uploads/` | `UPLOADS_DIR` |
| `cdn/files/` | `Data/files/` | `CDN_FILES_DIR` |
| `cdn/posters_combined/` | `Data/posters_combined/` | `CDN_POSTERS_DIR` |
| `instance/` | `Data/instance/` | `INSTANCE_DIR` |
| `backups/` | `Data/backups/` | `BACKUPS_DIR` |
| `logs/` | `Data/logs/` | `LOGS_DIR` |
| `sqlite:///amanflix_db.db` | `Data/instance/amanflix_db.db` | `DB_URI` |

## Typical Update Workflow

```
1.  Download/copy the new code to Z:\Amanflix\Amanflix_new_version\
2.  Open a terminal on the production server
3.  cd Z:\Amanflix
4.  python update_helper.py update
5.  Confirm with 'y'
6.  Wait ~30 seconds
7.  Done — app is running with the new code
```

If the new version has problems:

```
python update_helper.py rollback
```

## Troubleshooting

### "update_config.json not found"
Run `python update_helper.py configure` first.

### "New version folder not found"
Place the new code in `Amanflix_new_version/` before running update.

### App doesn't start after update
- Check `python update_helper.py status` — look for `[MISS]` paths
- Make sure `data_config.json` was written (status will show it)
- Check if venv was restored: `Amanflix_prod\api\venv\Scripts\python.exe` should exist
- Try starting manually: `cd Amanflix_prod\api && venv\Scripts\python.exe app.py`

### New version needs additional Python packages
After the update completes:
```batch
cd Z:\Amanflix\Amanflix_prod\api
venv\Scripts\pip.exe install -r requirements.txt
```
Then restart:
```batch
venv\Scripts\python.exe app.py
```

### Backup folder getting too large
Old backups don't contain data files (those stay in `Data/`), but they do contain code + any leftover files. Manually delete old backups from `Amanflix_backups/` to free space.

### venv becomes corrupted or incompatible
Recreate it:
```batch
cd Z:\Amanflix\Amanflix_prod\api
python -m venv venv
venv\Scripts\pip.exe install -r requirements.txt
```
